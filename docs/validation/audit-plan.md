## Categories in scope

1. **Redundant codification** — same rule stated in 2+ places, or multi-step protocol where one step would fire the same outcome.
2. **Process weight that doesn't earn its cost** — checks/gates/sub-steps whose failure mode is rare or whose value is marginal vs. token cost.
3. **Premature abstraction** — structure built for hypothetical future use cases.
4. **Defensive scaffolding** — error handling/fallbacks for scenarios that can't occur given upstream guarantees.
5. **Verbose/wordy language** — load-bearing rule earns its place but the prose around it is bloated.

(Original category "derivable scaffolding" dropped — every concrete case folds into #1 or #3.)
